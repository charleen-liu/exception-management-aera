SAP Business Network · Supply Chain Monitor · Exception Agent
Detailed User Story — Current State & Agent-Driven Flow
How a buyer manages a location-based supply constraint across Forecast, Order Confirmation, and ASN — before and after the SCM Exception Agent.
Version 2.0 · March 2026 Updated from Deliverable 2 — Agent Flow Diagram Volterra Components / Plant US01
Persona
Sarah, Supply Chain Buyer / Planner — ACME Inc. (US10)
Scope
10–50 buyer locations · Thousands of planning parts · 60 active POs across 15 suppliers
Supplier
Volterra Components GmbH
Vendor #5000
Planning Model
Constraint-based: supplier forecast commits = available capacity per location

Scenario Trigger

Sarah's planning system (Purchasing Org: NV00, Group: N01) publishes a rolling, location-specific forecast to Volterra Components GmbH via SAP Business Network. Volterra responds with partial commits:

Plant US01 (Chicago) — 60% · Weeks 4 & 6 · CRITICAL
Plant US02 (Dallas) — 80% · Week 6 · MEDIUM

A commit shortage alert appears in the Forecast Alerts tile. In the planning model, committed quantities represent the maximum available supply per location.

Stage 1 — Forecast Location-Based Capacity Signal
1
Sarah publishes forecast to supplier via SAP Business Network
Buyer
Current State

Sarah publishes a rolling, location-specific forecast to Volterra via SAP Business Network. No agent involvement at this step.

Agent-Driven

Same buyer action. The agent begins monitoring for supplier responses as soon as the forecast is published — ready to act the moment Volterra commits.

2
Supplier responds with partial commits · Agent detects and aggregates
SupplierAgent
Current State

Volterra's partial commits appear in the Forecast Alerts tile as 19 flat, unranked rows — no location grouping, no severity, no root cause. Sarah opens individual records one by one to identify affected locations and critical weeks. She tracks the Plant US01 shortage in an external spreadsheet.

  • No prioritisation by location impact — buyers must infer which plants are most constrainedMolex, Richemont, PepsiCo, Idexx
  • No global supplier + location filter simultaneouslyIR #282119; Range/Multi Value Selection IR
  • 19 rows with no grouping, ranking, or aggregationMolex, Richemont, PepsiCo, Idexx
Agent-Driven

The SCM Agent immediately detects Volterra's partial commits and aggregates all part-level shortfalls by plant into a ranked plant-level gap summary:

Plant US01 (Chicago): 60% commit from Volterra across 12 parts, Weeks 4 & 6 — CRITICAL Plant US02 (Dallas): 80% commit from Volterra Week 6 — MEDIUM

19 rows collapse into 2 ranked plant-level signals. No spreadsheet. No manual row review. The agent simultaneously drafts a supplier root cause enquiry.

3
Agent surfaces prioritised alert summary and drafts supplier outreach
Agent
Current State

Does not exist. Sarah has no prioritised view until she manually reviews each row. She composes a root cause enquiry email to Volterra herself — outside the system, with no record linkage.

  • Root cause and location-specific capacity rationale not visible in the systemPepsiCo
  • Supplier response arrives via email and is never recorded in the planning systemPepsiCo
Agent-Driven

The agent surfaces the prioritised plant-level alert summary in SCM Monitor before Sarah opens it. When she arrives, she sees one ranked insight card — not a flat list.

The agent has pre-drafted the supplier root cause message, pre-populated with plant, affected weeks, commit percentages, and PO references. Ready for Sarah's review.

4
Buyer opens SCM Monitor
Buyer
Current State

Sarah opens the Forecast Alerts tile and begins manual triage: filtering, clicking into rows one by one, cross-referencing locations, building a picture of the constraint in a spreadsheet. Typically 10–20 minutes before any action is taken.

Agent-Driven

Sarah opens SCM Monitor. The agent has already surfaced a prioritised, plant-level alert summary — no manual filtering needed. She sees immediately: which plants are constrained, severity ranking, weeks affected, and a pre-drafted supplier message ready for her review.

Time to first action: seconds, not minutes.

5–6
Buyer reviews pre-drafted supplier message and approves send (one click)
Buyer
Current State

Sarah composes an email to Volterra from her email client, requesting a root cause explanation. No system record is created. She waits approximately one day for a reply.

  • No in-system record of supplier communication
  • No automated follow-up if supplier does not respond within SLA
  • Supplier response captured in inbox only — never linked to the alert
Agent-Driven

Sarah reviews the pre-drafted supplier message in SCM Monitor. She edits if needed and approves the send with one click. The message is logged against the exception record with timestamp and status — in-system, fully traceable.

7
Agent sends message · Supplier provides root cause · Agent classifies and links
AgentSupplier
Current State

Sarah's manually composed email reaches Volterra. ~1 day later, Volterra explains a raw material shortage impacting the Plant US01 production line. Sarah reads the reply in her inbox and manually updates her spreadsheet with the root cause. The system retains no record of this exchange.

Agent-Driven

Upon buyer approval, the agent sends the message and monitors for Volterra's response. Volterra receives the message and provides root cause (capacity constraint — raw material shortage at US01) and expected resolution date.

The agent reads the supplier response, classifies the root cause, and links it to the open exception record automatically. No spreadsheet update required.

Stage 2 — Order Confirmation Capacity Propagates to Orders
8
Supplier submits OCs · Agent processes, re-classifies, and groups into one OC situation
SupplierAgent
Current State

Sarah navigates to the OC Deviations screen — a separate view from Forecast Alerts, with no linkage to the upstream constraint. She manually filters by Volterra and receiving location. There are no delta columns for ordered vs. confirmed quantities. She manually calculates capacity loss per PO line. 27 deviations = 27 individual row-expand clicks.

  • No delta columns (ordered qty vs confirmed qty)Molex, Richemont, CocaCola
  • No aging indicators (days outstanding per deviation)Molex, Richemont, CocaCola
  • No location-level rollup of total supply gapMolex, Richemont, CocaCola
  • OC deviations not linked to upstream forecast shortfall—
Agent-Driven

Volterra submits OCs reflecting available capacity. The agent immediately:

1) Identifies forward-looking supply constraints from the OC submissions
2) Re-classifies the missed commit as capacity-constrained, using root cause already captured from Volterra's earlier response
3) Flags the situation as likely impacting upcoming POs at Plant US01
4) Groups all near-term PO lines from Volterra into a single consolidated OC situation

OC deviations are linked to the open Forecast Alert. Sarah does not navigate to a separate screen — the OC situation is presented in context of the constraint she already knows about.

9
Buyer reviews consolidated PO view · Approves batch · Escalates flagged exceptions
Buyer
Current State

Sarah approves one deviation and rejects the other without being able to see supplier reason codes, confirmed ship date changes, or whether partial shipments are planned. She opens multiple PO screens to piece together context before deciding. No batch approval exists — each deviation is handled individually.

  • Supplier context missing at point of decision — reason codes, ship dates, partial shipment plans require multi-screen navigationCocaCola
  • No batch approval for in-tolerance deviationsMolex
  • No AI-guided recommendation on approve/reject
Agent-Driven

Sarah sees one consolidated view of all Volterra / Plant US01 PO lines — pre-grouped, pre-analysed, with delta quantities, days outstanding, supplier reason code, and upstream forecast position visible inline at point of decision.

The agent has classified deviations: in-tolerance items are recommended for batch approval with rationale; out-of-tolerance or high-risk deviations are flagged for individual review.

Sarah approves the in-tolerance batch with one action. She reviews and escalates only the flagged exceptions. No multi-screen navigation. No manual delta calculation.

Stage 3 — ASN / Execution Physical Reality Confirms Capacity Loss
10
Agent monitors ASN submissions · Detects shortfall · Updates cumulative erosion trail
AgentSupplier
Current State

The ASN Alerts tile shows only aggregate totals: 14 missing · 10 late · 8 qty deviations · 6 date deviations. Sarah manually filters by Volterra and Plant US01, finds 1 missing ASN (due Wednesday) and 1 ASN qty deviation. She emails Volterra again — the 3rd email about the same root cause. She waits 2 days, logs back in to check, then manually reconstructs the full erosion story across 4+ screens and a spreadsheet.

  • ASN alerts not linked to upstream forecast or OC signals—
  • No push notification when supplier acts — 2-day manual check-back—
  • No cumulative erosion view (Forecast → OC → ASN) in one place—
  • Requested and confirmed ship dates not visible in ASN alert viewCustomer feedback
Agent-Driven

The agent continuously monitors ASN submissions against confirmed PO quantities. It detects the quantity shortfall immediately and updates the cumulative erosion trail across all three stages:

Plant US01 supply from Volterra — Weeks 4–6: Forecast: 100% (demand) OC Confirmed: 60% (capacity constraint) ASN Shipped: 45% (physical delivery)

No manual reconstruction. No 4+ screen navigation. The full story is maintained automatically in the exception record.

11
Buyer receives proactive ASN alert · Confirms physical supply loss
Buyer
Current State

Sarah discovers the ASN status by logging in two days after her last email. She finds the ASN was submitted at a quantity below confirmed (which was already below forecast). She updates her spreadsheet and escalates via email — spending 30+ minutes compiling the erosion data from three screens.

Agent-Driven

Sarah receives a proactive ASN alert pushed to her in SCM Monitor — no manual check-back required. The alert confirms physical supply loss at Plant US01 and presents the full Forecast → OC → ASN erosion trail inline.

Sarah confirms the supply loss in seconds. No separate ASN screen. No timeline reconstruction.

12
Agent identifies mitigation options · Drafts escalation summary with root cause and flex capacity
Agent
Current State

Sarah separately checks Plant US02 ASN status as a potential mitigation candidate — a manual step requiring a separate plant view. She then compiles an escalation email from scratch: supplier name, plant, affected weeks, commit percentages, linked PO numbers, and root cause. This takes 30+ minutes.

  • Plant US02 mitigation capacity is never surfaced proactively — requires manual cross-check—
  • Escalation drafting is fully manual — 30+ minutes of data assembly across 3 screens and a spreadsheet—
Agent-Driven

The agent proactively identifies mitigation options across plants and surfaces Plant US02 as a flex capacity candidate (80% commit, no OC deviations, ASN on track for Week 6) — without Sarah having to ask.

The agent drafts a complete escalation summary containing:

Root cause: Volterra raw material shortage, US01 line Erosion trail: Forecast 100% → OC 60% → ASN 45% Affected weeks: 4, 6 Linked POs: [PO numbers] Flex capacity: Plant US02 — 80% commit, Week 6 No OC deviations, ASN on track

Nothing needs to be compiled from scratch.

13
Buyer reviews agent-generated mitigation summary · Approves escalation
Buyer
Current State

Sarah sends the escalation email from her email client. No system record is created. The escalation is not linked to the exception, the forecast alert, the OC deviations, or the ASN. Her planning manager receives an email with no system traceability.

Agent-Driven

Sarah reviews the agent-generated mitigation summary and escalation draft. She approves the escalation with one action. The escalation is sent in-system, logged against the exception record with full traceability: timestamp, recipient, data snapshot, and linked source records (forecast alert, OC deviations, ASN).
